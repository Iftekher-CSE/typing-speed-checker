Local VS Session Storage
 Similarities:
1. Both are part of Web API and user for storing data.
2. Both store data as value against each key.
3. They store values as string.

Dissimilarities :
1. SessionStorage store data untill the browser is closed. LocalStorage stored data has no expiration.
2. SessionStorage store data is never transferred to server. LocalStorage data may transferred to server.
3. SessionStorage Data storage limit is larger than a cookie (at most 5MB). LocalStorage storage limit is the maximum amongst the two.

Global scope vs block scope
1. If a variable is declared outside all functions is considered as global scope. On the other hand a variable is declared within a function or curly braces ({}) is a block scope.
2. A variable declared globally can be accessed from any where of the code within a file. A variable declared locally, within function or within curly braces ({}), can be accessed within that function or curly braces.
3. variable declared with let and const follow this mentioned rules. but variable declared with var doesn't follow this rules, this is called hoisting.

Event loop in JS
JS execute it's code through event loop model. We know that JS is a single thread language, that means JS execute code one by one, can't execute multiple code at time. But in operation we can see that JS execute regular synchronous task like function callBack, loop etc. during asynchronous task like setTimeout, setInterval etc. is going on. This actually manage through event loop process.
For example, there are three console.log. When we run this code call stake console.log(a) and display result 10. when call stake found setTimeout(), it sent this function to WebAPI for count require 1sec. during this count on going in webAPI call stake execute console.log(b) and display result 20. after 1sec setTimeout() comes to event queue. in this situation event loop check call stake, if there is nothing to execute in call stake, event loop pass setTimeout() function in the call stake for execution and display result 200.

8 Ways to get undefine
1. If we want to console.log(); a variable that in not initialized will give an undefine.
2. If we want to console.log(); a function without explicit return will give undefine.
3 . If function parameter is not passed console.log(); will return undefine for that parameter.
4. If return form function in not explicitly mentioned will give undefine.
5. If we want to get a property that is not in the object, will give undefine.
6. If index number in more than the array elements, will give undefine.
7. If we want to get a deleted value from the array elements will give undefine.
8. If we set value of a variable as undefine, that variable will give undefine. [ for example: let a = undefine; ]
